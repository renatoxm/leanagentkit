## Procedure

### 1. Gather parameters
- Get the artifact **name** (ask if not given). Derive `<Name>`, `<name_kebab>`,
  `<name_snake>` from it.
- Ask each **per-generation prompt** in the recipe as ONE interactive set
  (e.g. roles). Use the host's multiple-choice UI when available.

### 2. Build the plan (do NOT write yet)
Read the recipe steps. Produce a concrete plan listing, with real resolved paths:
- files to **create** (path + 1-line purpose),
- files to **edit** (path + exactly what gets inserted + the anchor),
- i18n keys added per locale,
- the role/access decision.
Show this plan to the user and ask for approval. If the recipe seems stale
(referenced anchor/file not found), say so and offer to re-author via
`leanagentkit-skill-artifact-template`.

### 3. Generate on approval
Apply each step in order: render templates with the resolved params, insert
edits at their anchors (idempotent — skip if already present), add i18n stubs to
every listed locale.

### 4. Verify
Run the recipe's post-generation checks (typecheck/lint/route presence where
possible). Report what was created, what was edited, and any check that failed.

### 5. Record
Add a one-line note to `docs/memory/PROGRESS.md` (new <artifact-type> `<name>`)
and, if relevant, update `docs/CODEBASE_MAP.md`.

## Rules
- Never skip the plan/approval step.
- Edits are idempotent; re-running with the same name must not duplicate entries.
- If a value isn't in params/prompts and can't be derived, ask — don't guess.
